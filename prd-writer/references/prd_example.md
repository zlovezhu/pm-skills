# PRD 范例：人员管理

> 以下为一份完整的产品需求文档范例（"人员管理"功能模块）。
> 请以此作为写作风格、编号结构和内容细节度的参考标准。
> 范例选用通用 B 端"人员管理"模块，方便迁移到任何项目。

---

**1 搜索：**

1.1 支持模糊搜索成员名称；输入内容后回车或离开选框直接搜索；不区分大小写

**2 角色筛选：**

2.1 点击下拉框可筛选特定角色的成员；默认显示"全部角色"；可选项包括：全部角色、管理员、QA人员、研发人员、访客

**3 添加成员：**

3.1 点击右上角【+添加成员】按钮，弹出添加成员弹窗

3.2 弹窗包含三个Tab页签：手动添加、从组织架构选择、~~邀请链接~~（灰色，暂时不上）

**3.3 手动添加：**

（此处插入原型图）

  3.3.1 成员输入框：

    3.3.1.1 支持直接复制粘贴成员名称进行添加

    3.3.1.2 输入框下方显示小字提示："如果输入多个人名，请用分号隔开。如果用户已经存在，则将用户组作合并处理。"

    3.3.1.3 输入框支持多行输入

  3.3.2 分配角色：

    3.3.2.1 必填项，点击下拉框选择角色

    3.3.2.2 可选角色：管理员、QA人员、研发人员、访客

    3.3.2.3 默认显示"请选择角色"占位文字

  3.3.3 操作按钮：

    3.3.3.1 【取消】：关闭弹窗，不保存任何更改

    3.3.3.2 【确认添加】：验证必填项后提交；若未选择角色，高亮下拉框并提示"请选择角色"；添加成功后关闭弹窗并刷新列表

**3.4 从组织架构选择：**

（此处插入原型图）

  3.4.1 左侧组织架构树：

    3.4.1.1 展示企业组织架构层级，支持展开/折叠

    3.4.1.2 显示部门名称及部门人数（格式：部门名称(人数)）

    3.4.1.3 支持勾选部门下全部成员或单独勾选成员

    3.4.1.4 顶部显示"勾选部门/成员"提示文字

  3.4.2 右侧已选择面板：

    3.4.2.1 显示"已选择"标题及已选人数（格式：已选择 X人）

    3.4.2.2 列出所有已选成员的名称

    3.4.2.3 每个成员名称后有删除按钮（×），点击可移除该成员

  3.4.3 分配角色：

    3.4.3.1 必填项，点击下拉框选择角色

    3.4.3.2 可选角色：管理员、QA人员、研发人员、访客

  3.4.4 操作按钮：

    3.4.4.1 【取消】：关闭弹窗，不保存任何更改

    3.4.4.2 【确认添加】：验证必填项后提交；若未选择成员，提示"请选择成员"；若未选择角色，提示"请选择角色"；添加成功后关闭弹窗并刷新列表

~~3.5 邀请链接：（暂时不上）~~

（以下为灰色内容，表示暂不上线）

（此处插入原型图）

  3.5.1 链接展示区：

    3.5.1.1 展示一个只读的链接输入框，显示邀请链接地址

    3.5.1.2 链接格式示例：https://example.com/invite?token=...

  3.5.2 【复制链接】按钮：

    3.5.2.1 点击后将链接复制到剪贴板

    3.5.2.2 复制成功后显示toast提示"链接已复制"

    3.5.2.3 可将链接发送给非项目成员，点击链接后跳转至申请加入项目页面

  3.5.3 分配角色：

    3.5.3.1 必填项，用于指定通过此链接申请加入的成员默认角色

    3.5.3.2 可选角色：管理员、QA人员、研发人员、访客

  3.5.4 申请流程说明：

    3.5.4.1 非项目成员点击邀请链接后，跳转至申请加入页面

    3.5.4.2 用户提交申请后，需要管理员审批

    3.5.4.3 管理员审批通过后，该成员自动加入项目并分配指定角色

  3.5.5 操作按钮：

    3.5.5.1 【取消】：关闭弹窗

    3.5.5.2 【确认添加】：若未选择角色，高亮下拉框并提示"请选择角色"；选择角色后点击确认，生成带角色信息的邀请链接

**4 人员列表：**

（此处插入原型图）

4.1 列表字段：

  4.1.1 勾选框：

    4.1.1.1 每行数据前显示勾选框，用于批量操作

  4.1.2 成员：

    4.1.2.1 显示成员头像（圆形）和成员名称

    4.1.2.2 成员名称支持点击，点击后可跳转至成员详情页（如有）

    4.1.2.3 若无头像则显示默认头像图标

  4.1.3 角色：

    4.1.3.1 以标签形式展示成员角色

    4.1.3.2 不同角色使用不同颜色：管理员（红色背景）、研发人员（蓝色背景）、QA人员（绿色背景）、访客（灰色背景）

  4.1.4 加入时间：

    4.1.4.1 显示成员加入项目的日期

    4.1.4.2 格式：YYYY-MM-DD（如：2022-03-15）

  4.1.5 操作：

    4.1.5.1 编辑按钮：点击进入编辑成员角色状态

    4.1.5.2 删除按钮：点击删除该成员

4.2 批量操作：

  4.2.1 批量操作栏：

    4.2.1.1 当勾选一个或多个成员时，列表上方显示批量操作栏

    4.2.1.2 操作栏显示已选择的成员数量（格式：已选择 X 项）

    4.2.1.3 操作栏包含【批量编辑角色】和【批量删除】两个按钮

    4.2.1.4 当未勾选任何成员时，批量操作栏隐藏

  4.2.2 批量编辑角色：

    4.2.2.1 点击【批量编辑角色】按钮，弹出批量编辑角色弹窗

    4.2.2.2 弹窗标题："批量编辑角色"

    4.2.2.3 弹窗内容：显示"将 X 位成员的角色修改为："提示文字，X为已选成员数量

    4.2.2.4 角色下拉框：可选角色包括管理员、QA人员、研发人员、访客

    4.2.2.5 【取消】按钮：关闭弹窗，不保存更改

    4.2.2.6 【确认修改】按钮：若未选择角色，提示"请选择角色"；选择角色后批量更新所选成员的角色，成功后关闭弹窗、刷新列表、显示"批量修改成功"提示

  4.2.3 批量删除：

    4.2.3.1 点击【批量删除】按钮，弹出批量删除确认弹窗

    4.2.3.2 弹窗标题："批量删除确认"

    4.2.3.3 弹窗内容："确定要删除选中的 X 位成员吗？删除后这些成员将无法访问本项目。"，X为已选成员数量

    4.2.3.4 【取消】按钮：关闭弹窗，不执行删除

    4.2.3.5 【确认删除】按钮：执行批量删除操作，成功后关闭弹窗、刷新列表、取消所有勾选、显示"批量删除成功"提示

4.3 成员编辑：

  4.3.1 点击编辑按钮后，角色字段变为可编辑的下拉框

  4.3.2 可修改成员的角色（管理员、QA人员、研发人员、访客）

  4.3.3 修改后自动保存，无需额外确认

4.4 成员删除：

  4.4.1 点击删除按钮后，弹出二次确认弹窗

  4.4.2 弹窗内容："确定要删除该成员吗？删除后该成员将无法访问本项目。"

  4.4.3 【取消】：关闭弹窗，不执行删除

  4.4.4 【确认删除】：执行删除操作，成功后刷新列表并显示"删除成功"提示

4.5 列表分页：

  4.5.1 默认每页显示10条数据

  4.5.2 支持切换每页显示条数（10条/20条/50条）

  4.5.3 显示总数据量及当前页码信息

  4.5.4 切换页面时，取消所有勾选状态

4.6 空状态：

  4.6.1 当项目无成员时，列表区域显示空状态提示

  4.6.2 提示文字："暂无成员，请点击右上角添加成员"
