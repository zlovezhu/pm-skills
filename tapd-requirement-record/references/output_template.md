# 回执模板与完整示例

> 给 `tapd-requirement-record` Skill 用。需求和缺陷共用同一份格式，差异只在前缀图标和字段名。

## 一、模板原文

### 已落单（提交成功后）

```
✅ 已提单 · {单类型}
─────────────────────────
📌 {标题}
🏷️ 优先级：P{X}  ·  {分类|模块}：{XX}  ·  处理人：{XX}
🔗 TAPD：{链接}

⚠️ 我做了 {N} 处推断，便于你核对：
  · {字段1}：{值}（依据：{来源}）
  · {字段2}：{值}（依据：{来源}）

📎 附件：{日志/截图，如有}
```

### 待确认（草稿阶段）

```
🟡 待确认 · {单类型} 草稿
─────────────────────────
📌 {标题}
🏷️ 优先级：P{X}  ·  {分类|模块}：{XX}  ·  处理人：{XX}

📝 描述预览：
{描述前 2~3 行，超过省略号}

⚠️ 我做了 {N} 处推断 + {M} 处需要你补：
  · 推断：{字段}：{值}（依据：{来源}）
  · 待补：{字段}：[原因]

📎 附件：{日志/截图，如有}

确认后我就提交。
```

## 二、完整示例 1：需求（已落单）

输入：
> 「{某个功能模块}能不能加个按文件名搜索的功能？现在文件多的时候找不到」（来自团队聊天工具，发言人 {同事昵称}）

```
✅ 已提单 · 需求
─────────────────────────
📌 {功能模块}支持按文件名模糊搜索
🏷️ 优先级：P2（Middle）  ·  分类：{功能模块}  ·  处理人：[研发-XX]
🔗 TAPD：https://tapd.example.com/xxx/prong/stories/view/1234567

⚠️ 我做了 4 处推断，便于你核对：
  · 标题：从"按文件名搜索的功能"提炼（依据：原话）
  · 分类：{功能模块}（依据：原话提到"{功能模块}"）
  · 优先级：Middle（依据：未提及紧急程度，按默认）
  · 提出人：{同事昵称}（依据：聊天发言人）
```

## 四、模板使用守则

1. **行数硬上限**：已落单回执 ≤ 8 行（不含推断列表）；待确认回执 ≤ 14 行（含描述预览）
2. **推断必列**：哪怕只推断了 1 个字段也要列；让用户一眼能查
3. **未填的选填项不列**：避免噪声
4. **附件路径用相对路径或文件名**：不要贴绝对路径（如 `C:\Users\<user>\...`），脱敏要求
5. **图标固定不换**：✅/🟡/📌/🏷️/🔗/⚠️/📝/📎 各司其职，便于用户视觉扫描时形成肌肉记忆
